# 產品需求文件 (PRD) - 讀書筆記本

## 1. 專案概述
### 背景與動機
學生在學習與課外閱讀過程中，常常會閱讀大量書籍。為了解決讀後容易遺忘的問題，並幫助學生有效整理知識與心得，我們需要一個個人專屬的數位讀書筆記本，方便日後回顧。

### 目標用戶
- 學生（從小學到大學等，有閱讀習慣與課業需求者）

### 核心價值主張
提供學生一個簡單直覺的介面，快速記錄書名、撰寫心得並給予評分，透過搜尋功能，讓閱讀過的書籍成為隨時可以檢索的知識庫。

## 2. 功能需求
以下為本系統的主要功能與對應的使用者故事：

1. **紀錄書籍資料**
   - 作為 **學生**，我希望 **能新增我正在讀或讀完的書名**，以便 **建立我的個人書單**。

2. **撰寫讀書心得**
   - 作為 **學生**，我希望 **能為書籍寫作閱讀心得與重點筆記**，以便 **記錄當下的想法與整理書中知識**。

3. **給予書籍評分**
   - 作為 **學生**，我希望 **能針對讀完的書籍給予 1 到 5 星的評分**，以便 **標記這本書對我的價值與推薦程度**。

4. **搜尋書籍與筆記**
   - 作為 **學生**，我希望 **能透過書名或關鍵字搜尋我建立的書籍**，以便 **在需要參考時快速找到對應的心得**。

5. **編輯與刪除紀錄**
   - 作為 **學生**，我希望 **能隨時修改之前寫的筆記或刪除不需要的書單**，以便 **保持筆記本內容的正確性與整潔**。

## 3. 非功能需求
### 技術限制
- **後端框架**：使用 Python 語言搭配 Flask 輕量級框架。
- **前端渲染**：使用 Flask 內建的 Jinja2 模板引擎進行 HTML 渲染，搭配基礎 CSS 美化介面。
- **資料庫**：使用 SQLite 關聯式資料庫，方便本地端開發與管理。

### 效能與安全考量
- 效能：確保搜尋功能可以在 1 秒內回傳結果，維持良好的使用者體驗。
- 安全：對使用者輸入的心得內容進行跳脫處理，防範基本的 XSS 攻擊。

## 4. MVP 範圍
為確保專案能順利推進並完成核心價值，功能優先級規劃如下：

### Must Have (必要功能 - MVP)
- 新增書名與書籍紀錄。
- 針對書籍撰寫心得。
- 給予書籍 1-5 星評分。
- 顯示書籍與心得列表。

### Should Have (重要功能)
- 透過書名進行基本搜尋功能。
- 編輯與刪除已有書籍與心得紀錄。

### Nice to Have (加分功能)
- 筆記支援簡單的 Markdown 語法排版。
- 依據評分高低排序書籍列表。

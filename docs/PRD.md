# 產品需求文件 (PRD) - 讀書筆記本

## 1. 專案概述
### 背景與動機
在資訊爆炸的時代，人們閱讀了許多書籍，但往往過了一段時間就忘記書本的內容與重點。因此，需要一個個人化的數位讀書筆記本，幫助使用者記錄閱讀軌跡、整理讀書心得，並能快速回顧與搜尋過去的閱讀紀錄。

### 目標用戶
- 喜愛閱讀，希望記錄個人閱讀歷程的人
- 需要整理重點摘要以便日後複習的學生或專業人士
- 想追蹤自己每年閱讀目標達成率的書蟲

### 核心價值主張
提供一個簡單、直覺且專屬的閱讀紀錄空間，讓每一本讀過的書都能轉化為個人知識庫的一部分，並透過系統化的整理提升閱讀效益。

## 2. 功能需求
以下為本系統的主要功能與對應的使用者故事：

1. **書籍管理 (新增/編輯/刪除書籍)**
   - 作為 **使用者**，我希望 **能新增書籍資料（書名、作者、封面圖片、分類）**，以便 **建立我的專屬書單**。
   - 作為 **使用者**，我希望 **能編輯或刪除已建立的書籍資料**，以便 **修正錯誤或移除不必要的紀錄**。

2. **閱讀狀態與進度追蹤**
   - 作為 **使用者**，我希望 **能標記書籍的閱讀狀態（未讀、閱讀中、已讀）及紀錄當前閱讀頁數**，以便 **隨時掌握自己的閱讀進度**。

3. **讀書筆記與心得紀錄**
   - 作為 **使用者**，我希望 **能為每本書撰寫多篇筆記或整體心得，並支援簡單排版**，以便 **保存閱讀時的靈感與重點摘要**。

4. **書籍評分與評價**
   - 作為 **使用者**，我希望 **能對已讀完的書籍給予星級評分（1-5顆星）及簡短短評**，以便 **日後回顧時能快速判斷該書的推薦程度**。

5. **書籍搜尋與過濾**
   - 作為 **使用者**，我希望 **能透過書名、作者或分類搜尋書籍，並能依據閱讀狀態過濾清單**，以便 **在大量藏書中快速找到我想看的書或筆記**。

## 3. 非功能需求
### 技術限制
- **後端框架**：使用 Python 語言搭配 Flask 輕量級框架。
- **前端渲染**：使用 Flask 內建的 Jinja2 模板引擎進行 HTML 渲染，搭配基礎 CSS 美化介面。
- **資料庫**：使用 SQLite 關聯式資料庫，方便本地端開發與輕量化部署。

### 效能與安全考量
- 效能：確保書籍列表頁面在資料量低於 1000 筆時，載入時間不超過 2 秒。
- 安全：過濾使用者輸入的內容（如筆記、心得），防範 XSS (Cross-Site Scripting) 攻擊與 SQL Injection。

## 4. MVP 範圍
為確保專案能順利推進並完成核心價值，功能優先級規劃如下：

### Must Have (必要功能 - MVP)
- 新增、編輯、刪除書籍基本資料。
- 顯示書籍列表頁面與單一書籍詳細頁面。
- 為書籍新增、檢視筆記與心得。
- 基本的閱讀狀態切換（未讀、閱讀中、已讀）。

### Should Have (重要功能)
- 書籍搜尋功能（依書名）。
- 星級評分功能。
- 閱讀進度紀錄（記錄頁數）。

### Nice to Have (加分功能)
- 透過 ISBN 或 Google Books API 自動帶入書籍資訊與封面。
- 閱讀數據統計（例如：今年已讀幾本書、各分類比例圖表）。
- 筆記支援 Markdown 語法。
